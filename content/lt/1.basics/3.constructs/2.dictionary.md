Jau mokame sąrašo duomenų struktūrą. Turėtumėte suprasti, kad ši duomenų struktūra pasižymi:

- indeksais grįstas priega prie elementų: norint gauti elementą, reikia nurodyti jo indeksą, kuris yra teigiamas sveikasis skaičius;
- tvarkos išlaikymu: kokia tvarka elementai įdėti, tokia tvarka galima juos ir pasiekti (naudojant indeksus);
- modifikuojama: sąrašą galima keisti pridedant prie jo, ištrinant elementus, keičiant juos (kai kurios duomenų struktūrų yra nekeičiamos);
- heterogeniškumu pagal tipus: į sąrašą galima įdėti įvairių tipų duomenis (`[1, "hello", 3.14, ["another", "list", 111.111]]`{ lang=python });

Tarkime turime sąrašą:

```python line-numbers=false
example_list = [22, -30, -40, 1, 100, 20000]
```

Sąrašą su jų indeksais nesunku atvaizduoti lentele:

| Indeksas | 0   | 1   | 2   | 3   | 4   | 5     |
| -------- | --- | --- | --- | --- | --- | ----- |
| Reikšmė  | 22  | -30 | -40 | 1   | 100 | 20000 |

Dažniausiai sąraše talpinamos vienodo tipo reikšmės (pavyzdyje sveikieji skaičiai). Skirtingų tipų reikšmes irgi galime talpinti, bet dažnai tai neturi didelės prasmės:

```python line-numbers=false
person_info = ["Vilius", 2000, 7 ,"Tauragė", "Kaunas"]
```

Turime informaciją apie žmogų. Tikriausiai, pirmasis elementas nurodo asmens vardą, antrasis gal gimimo metus, gal atlyginimą, gal mėgstamiausią skaičių, trečiasis gali gimimo dienos mėnesį, gal metus, kuriais pradėjo eiti į mokyklą ir t.t. Ši informacija nėra reikšminga, nes ji tiesiog susieta su indeksais:

| **Indeksas** | 0        | 1    | 2   | 3         | 4        |
| -------- | -------- | ---- | --- | --------- | -------- |
| **Reikšmė**  | "Vilius" | 2000 | 7   | "Tauragė" | "Kaunas" |

Būtų viskas aiškiau jeigu vietoje indeksų būtų etiketė, kintamojo pavadinimas, kuris paaiškintų prasmę:

| **Indeksas** | Vardas   | Bruto atlyginimas | Gimimo datos mėnesis | Gimtasis miestas | Gyvenamoji vieta |
| -------- | -------- | -------------------- | -------------------- | ---------------- | ---------------- |
| **Reikšmė**  | "Vilius" | 2000                 | 7                    | "Tauragė"        | "Kaunas"         |

Taip sužymėta informacija žymiai aiškesnė.

## Pirmasis žodynas

Paskutinėje esančią lentelę galima patalpinti į žodyno duomenų struktūrą. Kai kalbėsime apie kintamojo *indeksą* žodyne, vietojo indekso naudosime sąvoką **raktas**, kurį reikia suprasti, kaip reikšmės etiketę.

Jeigu norime sukurti žodyno reikšmę *Python* kalboje, reikia nurodyti **rakto-reikšmės** poras, jas apskliaudžiant riestiniais/figūriniais skliaustais `{}`{ lang=python }. Tai naudojantis lentele sukurkime žodyną:

```python line-numbers=false
person_info = {
  "name": "Vilius",
  "gross_salary": 2000,
  "birthday_month": 7,
  "hometown": "Tauragė",
  "residence_city": "Kaunas"
}

print(person_info)
```

Atspausdinus žodyną, gaunamos visos rakto-reikšmės poros:

```console
{'name': 'Vilius', 'gross_salary': 2000, 'birthday_month': 7, 'hometown': 'Tauragė', 'residence_city': 'Kaunas'}
```

Nors pavyzdyje visos raktai yra `str`{ lang=python } tipo, galime naudoti bet kokias primityvias (`int`{ lang=python }, `float`{ lang=python }, `str`{ lang=python } , `bool`{ lang=python }) reikšmes, kaip raktą:

```python line-numbers=false
mixed_keys = {
    4: "val_1",
    3.14: "val_2",
    True: "val_3",
    None: "val_4",
    "last_value": "val_10"
}

print(mixed_keys)
```

```console
{4: 'val_1', 3.14: 'val_2', True: 'val_3', None: 'val_4', 'last_value': 'val_10'}
```

```python line-numbers=false
student_record = {
    "name": "Kaja Rodowicz",

print(student_record)
```

## Prieiga prie elementų

Kaip ir su sąrašais, taip ir žodynais, mes galime pasiekti mums reikiamą konkrečią reikšmę. Elementų pasiekimas ir sintaksė analogiški:

- sąrašas: nurodomas **elemento indeksas** laužtiniuose skliaustuose;
- žodynas: nurodomas **reikšmės raktas** laužtiniuose skliaustuose;

Paimkime anksčiau sukurtą studento įrašo kintamąjį ir pabandykime pasiekti keletą reikšmių:

```python
student_record = {
    "name": "Kaja Rodowicz",
    "age": 22,
    "major": "Food Science",
    "courses": [
        {"course_name": "Food Chemistry", "grade": "10"},
        {"course_name": "Microbiology in Food", "grade": "9"},
        {"course_name": "Food Safety and Quality", "grade": "9"}
    ],
    "graduation_year": 2024,
    "contact": {
        "email": "kaja.rodowicz@example.com",
        "phone": "+48 987 654 321"
    }
}

student_name = student_record["name"]
student_age = student_record["age"]
student_major = student_record["major"]
print(f"Student name: {student_name}")
print(f"Student age: {student_age}")
print(f"Student major: {student_major}")
# Or we can print it directly without making a new variable
# print(f"Student name: {student_record["name"]}")
# print(f"Student age: {student_record["age"]}")
# print(f"Student major: {student_record["major"]}")
```

```console
Student name: Kaja Rodowicz
Student age: 22
Student major: Food Science
```

### Giliau esančių elementų pasiekimas

Galime pabandyti pasiekti `"contact"`{ lang=python } rakto reikšmę:

```python custom-line-numbering=27
# ...
student_contact=student_record["contact"]

# `student_contacts` has the type of dictionary
print(type(student_contact))

# Also it is same as this:
# student_contact = {
#     "email": "kaja.rodowicz@example.com",
#     "phone": "+48 987 654 321"
# }

# Now we can access the deepier values:
print(f"Student email: {student_contact['email']}")
print(f"Student phone: {student_contact['phone']}")
```

```console
<class 'dict'>
Student email: kaja.rodowicz@example.com
Student phone: +48 987 654 321
```

Kad nereiktų kurti naujų kintamųjų, mes galima, kaip su sąrašais sąraše, giliau esančius elementus galime pasiekti rašant dvigubą indeksavimą:

```python line-numbers=false
print(f"Student email: {student_contact['contact']['email']}")
print(f"Student phone: {student_contact['contact']['phone']}")
```

Galime pabandyti pasiekti ir sąrašą, kuris savyje turi žodynus:

```python custom-line-numbering=43
# ...
second_course_grade = student_record["courses"][1]['grade']
print(f"Second course grade: {second_course_grade}")

# student_record["courses"] is a list of dictionaries
# [
#     {"course_name": "Food Chemistry", "grade": "10"},
#     {"course_name": "Microbiology in Food", "grade": "9"},
#     {"course_name": "Food Safety and Quality", "grade": "9"}
# ]

# student_record["courses"][1] is the first element of a list of dictionaries 
# {"course_name": "Microbiology in Food", "grade": "9"}
```

```console
Second course grade: 9
```

### Neegzistuojantys raktai

Padarykime studento įrašą paprastesniu:

```python
student_record = {
    "name": "Kaja Rodowicz",
    "age": 22,
    "major": "Food Science",
    "graduation_year": 2024,
}
```

Ir pabandykime atspausdinti nurodyti raktą, kuris neegzistuoja žodyne:

```python
print(student_record["nationality"])
```

Įvykdę šią eilutę pamatysite įvykusią klaidą, kuri sako, kad nurodyto rakto nėra:

```console
KeyError: 'nationality'
```

Nurodydami raktą, turite būti įsitikinę, kad jis egzistuoja.
